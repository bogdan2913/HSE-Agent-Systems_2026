# Skill: Генерация системного промпта

## Требуемые инструменты
- read_report (локальный)
- save_report (локальный)

## Когда активировать
Этот skill активируется, когда:
- Аналитический отчёт (навык `analysis_report`) сформирован
- Пользователь просит сгенерировать новый системный промпт для конкретной задачи
- Нужно применить лучшие практики из отчёта к новому промпту

## Протокол действий

### Шаг 1. Прочитай аналитический отчёт
Вызови `read_report(name="analysis_report")`.
Если отчёт не найден — сообщи что сначала нужно запустить `analysis_report`.
Из отчёта извлеки:
- Лучшие практики
- Рекомендации для нового промпта
- Топ-паттерны из топ-промптов

### Шаг 2. Определи назначение нового промпта
Из запроса пользователя определи:
- **Роль агента** — кем он должен быть
- **Задача** — что агент должен делать
- **Аудитория** — кто будет общаться с агентом
- **Ограничения** — что агент не должен делать

### Шаг 3. Сгенерируй промпт строго по шаблону

Новый промпт ОБЯЗАН содержать все 5 критериев качества:

```
# Системный промпт: {название агента}

## Роль
{Чёткое определение роли агента — 1–2 предложения с указанием экспертизы и стиля}

## Задача
{Что агент делает — конкретно, без размытых формулировок}

## Инструкции по выполнению задачи
1. {Шаг 1 — конкретное действие}
2. {Шаг 2 — ...}
3. {Шаг 3 — ...}
...

## Примеры (Few-shot)

**Пример 1:**
Пользователь: {пример запроса}
Агент: {пример ответа в нужном формате}

**Пример 2:**
Пользователь: {пример запроса}
Агент: {пример ответа}

## Формат ответа
{Точное описание структуры ответа — Markdown / JSON / plain text, обязательные секции}

## Ограничения
- Никогда не {ограничение 1}
- Отклоняй запросы если {условие 2}
- Если {ситуация} — {что делать}
```

### Шаг 4. Проверь промпт по чеклисту

Перед сохранением убедись что в промпте есть:
- [ ] Явная роль с характером/стилем (Role/Persona)
- [ ] Минимум 2 примера вход/выход (Few-shot)
- [ ] Пошаговые инструкции по рассуждению (Chain-of-Thought)
- [ ] Конкретный формат вывода (Output Format)
- [ ] Минимум 3 ограничения (Guardrails)

Если что-то отсутствует — добавь перед сохранением.

### Шаг 5. Сохрани результат

Вызови `save_report(name="generated_prompt", content=...)` с полным текстом промпта.
После сохранения выведи:
```
✅ Промпт сгенерирован и сохранён.
Оценка по критериям:
- Role/Persona: присутствует
- Few-shot: {N} примеров
- Chain-of-Thought: присутствует
- Output Format: присутствует
- Guardrails: {N} ограничений
```
