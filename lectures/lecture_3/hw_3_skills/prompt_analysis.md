# Skill: Анализ системного промпта

## Требуемые инструменты
- get_file_contents (GitHub MCP)
- save_analysis (локальный)

## Когда активировать
Этот skill активируется, когда нужно:
- Разобрать системный промпт на составные части
- Оценить качество и полноту промпта
- Выявить приёмы prompt engineering в промпте
- Сохранить структурированный анализ для последующего использования

## Протокол действий

### Шаг 1. Получи содержимое промпта
Если промпт ещё не загружен — вызови `get_file_contents` с нужным путём к файлу в репозитории.
Репозиторий: `asgeirtj/system_prompts_leaks`.

### Шаг 2. Определи базовую структуру промпта
Выдели следующие блоки (если присутствуют):
- **Вводный блок** — первые строки, задающие контекст
- **Секции с инструкциями** — именованные блоки (## Header, XML-теги, нумерованные пункты)
- **Примеры** — секции с few-shot примерами
- **Ограничения** — что агенту запрещено делать
- **Формат вывода** — как оформлять ответы

### Шаг 3. Оцени промпт по 5 критериям

Для каждого критерия выставь оценку от 0 до 3:
- **0** — критерий отсутствует
- **1** — присутствует слабо / неявно
- **2** — присутствует, но можно улучшить
- **3** — реализован хорошо и явно

#### Критерий 1: Role / Persona
Есть ли у агента чёткая роль или персона?
- Примеры: "You are a helpful assistant", "Ты опытный аналитик данных"
- Проверь: задана ли роль явно, есть ли у неё характер/стиль/экспертиза

#### Критерий 2: Few-shot Examples
Есть ли в промпте конкретные примеры вход/выход?
- Примеры: блоки "User: ... Assistant: ...", секции Example, XML-примеры
- Проверь: сколько примеров, насколько они репрезентативны

#### Критерий 3: Chain-of-Thought / Reasoning
Инструктирует ли промпт агента думать пошагово?
- Примеры: "Think step by step", "First analyze, then...", "Let's think through this"
- Проверь: есть ли явные инструкции по процессу рассуждения

#### Критерий 4: Output Format
Задан ли формат вывода явно?
- Примеры: JSON-схема, Markdown-шаблон, "отвечай в формате...", XML-структура
- Проверь: насколько конкретно описан ожидаемый формат

#### Критерий 5: Guardrails / Constraints
Есть ли ограничения на поведение агента?
- Примеры: "не выдумывай", "откажись если...", "никогда не...", списки запрещённых тем
- Проверь: насколько полно покрыты граничные случаи

### Шаг 4. Сформируй вывод строго по шаблону

Вывод должен быть ТОЛЬКО в следующем формате — не добавляй ничего лишнего:

```
# Анализ промпта: {имя файла}

## Источник
- Файл: {путь к файлу}
- Размер: {количество символов} символов

## Структура промпта
- Выявленные секции: {список секций через запятую}
- Стиль разметки: {plain text / Markdown / XML / смешанный}
- Язык: {язык промпта}

## Оценка по критериям

| Критерий | Оценка (0–3) | Комментарий |
|---|---|---|
| Role / Persona | {0-3} | {1 предложение} |
| Few-shot Examples | {0-3} | {1 предложение} |
| Chain-of-Thought | {0-3} | {1 предложение} |
| Output Format | {0-3} | {1 предложение} |
| Guardrails / Constraints | {0-3} | {1 предложение} |

**Итоговая оценка: {сумма}/15**

## Сильные стороны
- {пункт 1}
- {пункт 2}

## Слабые стороны
- {пункт 1}
- {пункт 2}

## Ключевые паттерны
{3–5 конкретных приёма или особенности, которые выделяют этот промпт}
```

### Шаг 5. Сохрани результат

Вызови `save_analysis(name, content)`:
- `name` = имя файла без расширения и пути (например: `claude_system_prompt`)
- `content` = полный текст анализа из Шага 4

После успешного сохранения сообщи: "Анализ {name} сохранён."
НЕ добавляй ничего лишнего — только это сообщение.
